## 03 검색 알고리즘 


#### 선형 검색 (linear search)
```
직선 모양(선형)으로 늘어선 배열에서 검색하는 경우에 원하는 키값을 가진 원소를 찾을 때까지 맨 앞부터 순서대로 검색하는 알고리즘 
```

##### 보초법 (Sentinel Method)
```
- 선형 검색 알고리즘에서 루프를 벗어나기 위해 두 가지 조건이 있다.

* 선형 검색의 종료 조건 
  1) 검색할 값을 찾지 못하고 배열의 맨 끝을 지나간 경우 
  2) 검색할 값과 같은 원소를 찾은 경우 

- 이로 인해 선형 검색에서는 필연적으로 두개의 if문으로 인한 cost가 발생함 (비효율적)
- 그래서 이 비용을 반으로 줄이는 방법이 '보초법'이다. 
- 보초법은 검색하고자 하는 값을 배열의 가장 마지막에 추가한 뒤 검색을 진행하는 방법이다.
- 배열의 마지막에 검색 값이 추가되어 루프를 벗어나느데 문제가 없다.
- 대신 검색값을 발견하는 위치가 배열의 마지막인 경우 값이 없다는 것을 처리해주면 된다.

```

#### 이진 검색(binary search)
```
- 이진 검색은 선형검색보다 빠르게 검색할 수 있음 
- 이진 검색 알고리즘을 사용하려면 배열 데이터가 정렬(sort)되어 있어야 함 

* 이진 검색에서 검색 범위를 좁히는 과정 
1) a[pc] < key : 중앙(pc)에서 오른쪽으로 한칸 이동하여 새로운 왼쪽 끝 pl로 지정하고 검색범위를 뒤쪽 절반으로 줄입니다. (pc+1 ~ pr)
2) a[pc] > key : 중앙(pc)에서 왼쪽으로 한 칸 이동하여 새로운 오른쪽 끝 pr로 지정하고, 검색 범위를 앞쪽 절반으로 줄입니다. ( 0 ~ pc -1)

* 이진 검색의 종료 조건 

1) a[pc]와 key가 일치하는 경우 
2) 검색 범위가 더 이상 없는 경우 

```

#### 해시법(hasing)
```
- 데이터를 저장할 위치(=인덱스) 를 간단한 연산으로 구하는 것을 말함
- 원소의 검색 뿐만 아니라 추가/삭제도 효율적으로 수행가능 

* 용어 p132
- 해시값(hash value)
- 해시 테이블 (hash table)
- 해시 함수 (hash function) : 키를 해시값으로 변환하는 과정 ( 일반적으로 나머지를 구하는 연산 또는 그 연산을 응용할 때 주로 사용)
- 버킷 (bucket) : 해시 테이블에서 만들어진 원소
- 충돌(collision) : 저장할 버킷이 중복되는 현상 
  > 체인법 : 해시값이 같은 원소를 연결 리스트로 관리함
  > 오픈 주소법 : 빈 버킷을 찾을 때까지 해시를 반복함 
```

##### 체인법(chaining)
```
- 해시값이 같은 데이터를 체인 모양의 연결리스트로 연결하는 방법을 말함 
- 오픈 해시법(open hashing)이라고도 함 
```

##### 오픈주소법(open addressing)
```
- 오픈 주소법은 충돌 발생시 재해시(rehashing)을 수행하여 빈 버킷을 찾는 방법을 말함 
- 닫힌 해시법(closed hashing)이라고도 함 
- 충돌 발생시 빈 버킷이 나올 때까지 ㅣ재해시를 반복하므로 선형 탐사법(linear probing)이라고도 함 
```
